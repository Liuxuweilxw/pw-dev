《三角洲行动》陪玩拼单平台 软件需求说明书
一、项目概述
1.1 项目名称
《三角洲行动》陪玩拼单平台
1.2 项目背景
本平台为《三角洲行动》游戏玩家提供陪玩服务撮合、拼单组队的垂直服务平台，核心连接两类用户：
老板端：发起陪玩 / 拼单需求、支付费用的需求方用户
陪玩端：承接陪玩 / 拼单需求、获取佣金的服务方用户
平台提供房间创建、需求匹配、资金托管、订单结算等全流程服务，保障交易安全与用户体验。
1.3 核心目标
实现老板与陪玩的高效撮合，满足游戏陪玩、拼单组队需求
搭建安全的资金托管体系，保障双方资金安全
建立会员成长体系，提升用户留存与活跃度
完善举报与审核机制，维护平台合规环境
二、功能需求
2.1 登录与实名认证模块
2.1.1 登录功能
支持手机号 + 验证码登录：用户输入手机号，点击「获取验证码」，输入正确验证码后完成登录
后续迭代：完成实名认证后，支持手机号直接免验证登录
2.1.2 实名认证功能
强制触发逻辑：未实名用户点击「创建房间」时，弹出强制提示：「实名认证可保证资金安全，创建房间时必须先进行实名认证」
认证流程：
上传身份证正反面照片
上传手持身份证照片
手机号 + 验证码二次验证
认证通过后，解锁创建房间、资金交易等核心功能
2.2 老板端首页（需求发起端）
2.2.1 搜索与筛选
搜索功能：支持按陪玩昵称、房间关键词、游戏段位等维度搜索
筛选功能：提供筛选区，支持按价格区间、房间类型、陪玩等级、接单时效等条件筛选
2.2.2 创建房间功能
房间基础信息配置：
房间标识：支持自定义房间名称 / 标签
价格设置：
根据用户充值金额，自动展示对应可选择的游戏边数、场次
侧边栏可选择平台服务费率（10%/20%），系统自动计算剩余可分配给陪玩的总比例
人数限制：设置房间可加入的最大人数
备注：填写房间需求、陪玩要求、游戏时段等补充信息
资金冻结逻辑：用户确认下单后，支付对应金额，资金进入平台账户并冻结，待订单完成后完成结算
2.2.3 加入房间功能
房间列表展示：以卡片形式展示公开房间，包含信息：
房间等级标识、创建者昵称 / 等级、单价、加入按钮、出资比例、备注
待确认逻辑（需产品侧明确）：
出资比例定义：明确是「用户需出资的比例」还是「房主出资的比例」
排队逻辑：确认是否支持直接排队、发起排队、自动排队功能
2.2.4 房间详情功能
基础信息展示：
房间状态：实时展示「待加入 / 进行中 / 已完成」等状态
成员列表：展示房间内所有成员信息
空位展示：以座位样式直观展示剩余可加入空位
交互功能：
聊天区：支持房间内成员实时文字聊天
操作区：
生成邀请链接，可分享给好友
邀请腾讯生态好友（对接腾讯房间能力）
「确认完成」按钮：订单完成后点击，触发资金结算
「举报」按钮：可举报房间内违规用户
开桌逻辑：暂不支持实现在线开桌，房间以聊天室形式存在；用户加入后，对应资金自动转入房间账户
2.3 陪玩端首页（需求承接端）
2.3.1 搜索与筛选
搜索功能：支持按房间关键词、老板昵称、价格区间等维度搜索
筛选功能：提供筛选区，支持按价格、房间类型、需求紧急度等条件筛选
2.3.2 接单房间功能
房间列表展示：展示可接单房间，包含信息：房间单价、备注、剩余座位、出资比例、佣金
排序规则：优先展示按时间排序的 50% 常规需求，100% 大额需求排后展示（需明确「大额需求」判定标准）
房间详情弹窗：点击房间弹出详情面板，用户确认后进入房间
2.3.3 房间详情功能
基础信息展示：
房间状态：实时展示房间进度
成员列表：展示房间内老板、其他陪玩信息
空位展示：以虚位样式展示剩余可接单座位
交互功能：
聊天区：支持房间内实时文字聊天
举报功能：支持举报违规用户 / 房间
开桌逻辑：同老板端，暂不支持在线开桌，以聊天室形式承载
2.4 我的钱包模块
2.4.1 资产卡片
展示用户可用余额、积分余额，数据实时更新
2.4.2 充值功能
充值档位：预设5k/8k/12k/32k固定档位，同时支持自定义金额充值
充值渠道：支持支付宝、微信支付两种主流渠道
2.4.3 提现功能
提现中心：支持提现到原充值的微信 / 支付宝账户（需明确提现手续费、到账时间、限额等规则）
2.4.4 资金明细
展示全量资金流水，包含：
类型（充值 / 提现 / 订单消费 / 佣金结算）、金额、状态（成功 / 处理中 / 失败）、交易时间
2.5 个人中心模块
2.5.1 会员等级系统
等级与对应权益：
表格
等级	积分要求	权益
Lv1	0 积分	解锁新手房间功能
Lv2	500 积分	解锁限时房间功能
Lv3	2000 积分	解锁限时房间特效
后续迭代	-	收益回馈扩展权益
积分获取规则：用户每消费 1 元累计 1 积分，积分来源包含充值消费、订单消费、活动赠送等，累计计算
2.5.2 我的订单
订单列表展示：包含订单编号、成交人员、单价、出资比例、房间详情、订单进度
交互功能：
点击用户头像，可查看个人详情并发起举报
举报类型标准化：
骚扰与低俗内容
实物送货等违规行为
超时欠单
平台不受理的其他违规订单
举报处理逻辑：举报审核通过后，对应订单资金自动释放给房间（需明确审核时效与流程）
2.6 通用功能需求
2.6.1 资金托管与结算
资金流转逻辑：老板支付→平台冻结→订单完成→平台扣除服务费（10%/20%）→陪玩账户到账
异常处理：订单取消 / 举报通过时，资金原路退回老板账户
2.6.2 举报与审核系统
全平台覆盖举报入口：房间、订单、用户个人主页等场景
审核机制：建立人工 + AI 审核体系，保障平台合规
2.6.3 消息通知
订单状态通知、资金到账通知、举报处理通知等，支持 APP 推送 / 短信通知
三、非功能需求
3.1 性能需求
页面加载时间≤2s，高峰期并发支持≥1000 人
资金交易零延迟，数据一致性 100% 保障
聊天消息延迟≤1s
3.2 安全需求
实名认证信息加密存储，符合《个人信息保护法》等合规要求
资金交易采用加密链路，防篡改、防盗刷
聊天内容 AI 审核，拦截违规内容
3.3 兼容性需求
支持移动端（iOS/Android App）、H5、微信小程序多端适配
适配主流手机机型与系统版本
3.4 易用性需求
老板 / 陪玩角色区分清晰，操作流程简洁，提供新手引导
关键操作（如支付、提现）二次确认，避免误操作
3.5 可扩展性需求
预留多游戏接入接口，后续可扩展至其他射击类 / 竞技类游戏
预留会员权益、服务类型、支付渠道的扩展能力
四、待确认 / 待优化需求（产品侧需明确）
出资比例定义：明确房间列表中「出资比例」是用户需出资的比例，还是房主出资的比例
加入房间排队逻辑：确认是否支持直接排队、发起排队、自动排队功能
提现规则：明确提现手续费、到账时间、单日限额等规则
举报审核流程：明确举报审核时效、处理规则、资金释放的触发条件
大额需求判定标准：明确「100% 大需求」的金额阈值、订单类型定义
手机号登录迭代计划：明确后续手机号直接登录的上线时间与逻辑
五、角色定义
表格
角色	核心权限
老板用户	创建房间、加入房间、支付订单、发起举报、充值提现
陪玩用户	接单房间、进入房间、获取佣金、发起举报、充值提现
平台管理员	审核实名认证、处理举报、管理订单、配置会员权益、监控资金
