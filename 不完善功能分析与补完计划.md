# 项目功能不完善分析与补完计划

## 📊 总体评估

- **当前生产就绪度**: ~45%
- **项目阶段**: UI原型 + 前端流程演示

## 🔴 一级缺陷（阻断生产部署的功能）

### 1. 实名认证体系 - ⚠️ 占位符状态
**现状**: 仅有UI流程演示，无真实上传和审核
- `_showIdentityRequiredDialog()` - 显示认证说明文本
- `isVerified` 状态标记 - 点击"立即认证"直接模拟通过，无后端调用
- **缺失功能**:
  - [ ] 身份证正反面上传接口 (`POST /auth/verify/id-card`)
  - [ ] 手持身份证照片上传 (`POST /auth/verify/with-hand`)
  - [ ] 二次验证码验证 (`POST /auth/verify/sms-confirm`)
  - [ ] 认证状态查询接口 (`GET /auth/verify/status`)
  - [ ] 文件上传服务（七牛/OSS等）集成
  - [ ] 后端审核流程实现
  - [ ] 认证失败原因反馈

**建议**:
1. 后端实现认证相关API（3个新路由）
2. 前端集成文件选择器和上传组件（`image_picker` 插件）
3. 添加后端认证审核dashboard
4. 实现异步轮询查询认证进度
