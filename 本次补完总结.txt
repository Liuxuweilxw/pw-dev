     - 自动调用 _openRoomDetail(newRoom)

架构变化:
  ```
  API 改动:
  - abstract class PlatformApi
    - Future<void> createRoom() → Future<RoomItem> createRoom()

  - MockPlatformApi.createRoom()
    - 从只修改内部列表 → 返回已创建的房间
  
  - HttpPlatformApi.createRoom()
    - 从只发送请求 → 解析响应返回 RoomItem

  - _showCreateRoomDialog()
  ```

└─────────────────────────────────────────────────────────────────────────┘

新增文件:
  1. lib/src/services/chat_service.dart (180 行)
     - ChatMessage 数据模型 (与前端保持一致)
     - ChatService WebSocket 管理类
     - 连接/断开/重连机制
     - StreamController 消息流管理
     - 自动断线重连 (3秒间隔)
     - 资源清理 dispose() 方法

功能特性:
  ✓ WebSocket 连接管理 (ws://host/rooms/{roomId}/chat?token={token})
  ✓ 消息流 (Stream<ChatMessage>) for StreamBuilder 集成
  ✓ 连接状态流 (Stream<bool>) for UI 反馈
  ✓ 自动重连 (指数退避，最小 3 秒)
  ✓ 消息广播 (所有消息自动广播给房间所有用户)
  ✓ 历史消息 (预留接口，可调用 GET /rooms/{roomId}/messages)

修改清单:
  1. pubspec.yaml
     - 添加 web_socket_channel: ^2.4.0
  
  2. lib/src/ui/platform_shell.dart
     - 导入 ChatService
     - _openRoomDetail() 初始化 ChatService
     - 聊天区域改为 StreamBuilder<ChatMessage>
     - 发送消息改为调用 chatService.sendMessage()
     - 关闭时释放 ChatService 资源
     - 显示连接状态提示

UI 改进:
  - 聊天区域显示实时连接状态
    • "聊天室连接中..." (黄色提示，当前显示)
    • 支持实时消息显示 (连接成功后)
